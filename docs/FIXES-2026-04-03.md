# 修复记录 - 2026-04-03

## 问题修复

### 1. Java 版本升级
- **问题**: 项目需要 Java 21，但系统默认是 Java 8
- **解决**: 
  - 发现已安装的 JDK 21 位于 `C:\Program Files\Eclipse Adoptium\jdk-21.0.10.7-hotspot`
  - 更新 `mvnw.cmd` 设置正确的 `JAVA_HOME`

### 2. FunctionToolCallback.inputType 不能为 null
- **文件**: `src/main/java/demo/k8s/agent/config/DemoToolRegistryConfiguration.java`
- **问题**: `FunctionToolCallback.builder()` 需要指定 `inputType`
- **解决**: 添加 `.inputType(Map.class)` 
```java
return FunctionToolCallback.builder(toolName, (Map<String, Object> input) -> {...})
    .description(toolName)
    .inputType(Map.class)  // 新增
    .build();
```

### 3. ObjectMapper bean 缺失
- **文件**: `src/main/java/demo/k8s/agent/config/AgentConfiguration.java`
- **问题**: `FileSystemConversationRepository` 需要 `ObjectMapper` bean
- **解决**: 添加 ObjectMapper bean
```java
@Bean
public ObjectMapper objectMapper() {
    return new ObjectMapper();
}
```

### 4. Controller 路径冲突
- **文件**: `src/main/java/demo/k8s/agent/web/DemoChatController.java`
- **问题**: `DemoChatController` 和 `StatefulChatController` 都映射到 `/api/chat`
- **解决**: 将 `DemoChatController` 路径改为 `/api/internal`，并添加 `@Profile("dev")` 限制

### 5. 配置文件重复 key
- **文件**: `src/main/resources/application-local.yml`
- **问题**: 有两个 `demo:` 块
- **解决**: 合并为单个 `demo:` 块
